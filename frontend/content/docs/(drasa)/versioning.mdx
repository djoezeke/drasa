title: Versioning
description: How Drasa versions releases using Semantic Versioning.
---

Drasa follows Semantic Versioning in the format `<major>.<minor>.<patch>`.

## Rules

- Breaking changes increment **major**
- Backward-compatible features increment **minor**
- Fixes and maintenance updates increment **patch**

## Example progression

`1.0.0-alpha < 1.0.0-beta < 1.0.0-rc.1 < 1.0.0 < 1.1.0 < 1.1.1`

## Why this matters

This policy helps teams estimate upgrade impact and plan deployments safely.

Learn more at [semver.org](https://semver.org/).
